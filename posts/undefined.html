线程学习

谷科阳2025-04-152025-04-15

什么是进程？

OS中并发执行的多个程序

进程的特点

宏观并行，微观串行

原理:CPU会划分出若干个时间片,拥有时间片的程序才能执行,且一个时间片只能被一个程序拥有.由于时间片之间的执行是交替的,所以多个程序间也是交替执行，由于时间片的划分足够细小，交替频率足够快，就会形成并行的假象，但实际仍是串行

什么是线程？

是进程中并发执行的程序任务。

线程的特点：宏观并行，微观串行

原理:当程序拥有时间片后,其内部的多个线程至多有一个拿到该时间片,未拥有时间片的线程无法执行,所以其也是串行状态.由于时间片划分和交替的特征导致了线程之间并行的假象

程序是CPU最小的资源分配单位

线程是CPU最小的调度单位

线程的组成

执行所需的 时间片有CPU调度分配线程自身争夺而来

内存特征:栈独立而堆共享

所有线程都拥有自己的栈,但是使用的为同一个堆

线程的创建

1.相关的API

java.lang.Thread 线程类 实现Runnable接口

java.lang.Runnable 线程任务接口，拥有线程执行方法run()

2.两种创建方式

（1）继承Thread ，创建子类对象

优点: 身为独立的类,可以扩展独有内容

缺点:

1.必须绑定特定的任务内容,不够灵活

2.无法扩展父类,父类内容的扩展受限!

直接创建Thread对象，构造中传入Runnable的实现类对象、

Thread 引用名 = new Thread(Runnable实现类对象)

注:线程对象的执行为一次性，其内部任务大概率只会执行一次，并且Runnable接口为函数式接口，所以可以直接在构造中使用匿名内部类|lambda表达式对其简化

优点：书写简单，更加灵活，将线程对象与任务对象分离，便于管理和阅读

线程的使用

主函数也是主线程，其一定为首次获取时间片的线程,其他线程都是由主线程创建

开启多线程之后，JVM进程执行结束的标志会从主线程执行结束转换为所有线程执行结束

编程对象引用名字.start()开启线程

执行流程: 当线程开启后,会立即开始争抢时间片,若抢到时间片则执行自身 run()内容,如果丢失时间片则暂停执行,回到争夺队列,直至 run()内容执行完毕,线程才会退出争夺队列进入死亡

对应的为线程的生命周期

首先是新建状态。当我们使用 new Thread() 创建一个线程实例时，它就处于新建状态。这时候线程仅仅是在内存里构建出来了，还没开始运行，系统也没给它分配执行所需的资源。

接着调用 start() 方法，线程就进入可运行状态。这意味着线程准备好了，可以随时被 CPU 调度执行。不过可运行状态并不代表线程此刻就在运行，它有可能在等待 CPU 分配时间片，多个处于可运行状态的线程会竞争 CPU 资源。

一旦 CPU 调度选中了某个处于可运行状态的线程，这个线程就进入运行状态，开始执行 run() 方法里的代码。但要注意，因为 CPU 调度非常快，我们实际很难精准捕捉到线程处于运行状态的瞬间。

运行过程中，线程可能会进入阻塞状态。比如说，当线程尝试进入一个被其他线程占用锁的同步代码块，它就会进入阻塞状态，一直等到获取到锁才行。又或者当线程执行 I/O 操作，比如读取文件但数据还没准备好的时候，也会进入阻塞状态。

还有一种特殊的阻塞状态叫等待状态。像调用 Object 的 wait() 方法，线程会释放持有的锁，然后一直等待，直到其他线程调用同一对象的 notify() 或者 notifyAll() 方法，或者这个等待的线程被中断，才会结束等待状态。

另外，线程还可能进入超时等待状态。这种状态和等待状态类似，不过它设置了一个超时时间。像调用 Thread.sleep(long millis) 方法，线程就会在指定的时间后自动回到可运行状态。

最后，当线程的 run() 方法正常执行完，或者因为没捕获到异常提前终止，线程就进入终止状态。到了终止状态，线程就不再执行了，系统会回收它占用的资源。

总之，了解线程生命周期的这些状态，对我们编写稳定、高效的多线程程序特别重要，能帮我们避免像死锁、资源竞争这类问题。

阻塞等待状态

sleep-休眠

Thread.sleep(long 毫秒数);//1s = 1000ms

作用：使当前线程进入休眠状态，此时线程会释放自身时间片休眠过程中进入到有限期等待，直至休眠结束，才能回到就绪状态

需要处理非运行时异常，·run·不支持上抛异常，必须try-catch处理

常用于定时任务

join-插队

线程对象.join();

作用：使调用者线程插队到当前线程之前执行，当前线程会进入无限期等待状态,直至调用者线程执行结束进入死亡, 当前线程才能回到就绪状态

需要处理非运行时异常,在 run()中必须try-catch 处理

常用于调整线程执行优先级

sleep和join的区别：

1.sleep使进程进入的是有限期等待状态，join使线程进入无限期等待状态

2.sleep是静态方法通过Thread调用，join实例方法通过线程对象调用

3.sleep只操作自身线程，join一定影响其他线程
